template<typename T , typename TF >
bool celma::common::RangeStringIterator< T, TF >::operator== ( const RangeStringIterator< T, TF > &  other) const

Returns if the two iterators point to the same position.


Not a really foolproof check for equality, since the position is updated only after a sub-expression is processed. If a more meaningful test for equality would be needed, the current value of the iterator could be checked too.
